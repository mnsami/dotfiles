# /review

Review the provided code or changes for:

1. **Correctness**: Logic errors, edge cases, error handling
2. **Security**: Input validation, injection risks, auth issues
3. **Performance**: Obvious inefficiencies, N+1 queries, memory leaks
4. **Maintainability**: Readability, complexity, duplication
5. **Tests**: Coverage gaps, test quality

Provide feedback organized by severity (blockers, suggestions, nitpicks).
